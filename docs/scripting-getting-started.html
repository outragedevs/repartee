Scripting — Getting Started

Repartee supports Lua 5.4 scripting (via mlua) with a rich API modeled after WeeChat and irssi.

Script location

Scripts live in ~/.repartee/scripts/ as .lua files. All scripts in this directory are loaded automatically on startup.

Script format

Every script has two parts: a meta table and a setup function:

meta = {
    name = "hello",
    version = "1.0",
    description = "A simple greeting script",
}

function setup(api)
    api.on("irc.join", function(ev)
        local our_nick = api.store.our_nick()
        if ev.nick ~= our_nick then
            api.irc.say(ev.channel, "Welcome, " .. ev.nick .. "!")
        end
    end)

    api.log("Hello script loaded!")

    -- Return a cleanup function (optional)
    return function()
        api.log("Hello script unloaded")
    end
end

meta table

Field Type Required Description
name string Yes Script name (used for loading/unloading)
version string No Version string
description string No Short description

setup(api) function

Called when the script is loaded. Receives the api object with all scripting capabilities. Can optionally return a cleanup function that runs on unload.

Loading scripts

/script load hello
/script unload hello
/script reload hello
/script list

Autoloading

All .lua files in ~/.repartee/scripts/ are loaded automatically on startup. You can also explicitly list scripts in your config:

[scripts]
autoload = ["hello", "urllogger"]

Sandboxing

Scripts run in a sandboxed Lua environment. The following globals are removed for security:

  • os — no filesystem/process access
  • io — no file I/O
  • loadfile, dofile — no arbitrary file execution
  • package — no module loading

The following standard libraries are available:

  • math — math functions and math.random
  • string — string manipulation and patterns
  • table — table operations (insert, remove, sort, concat, etc.)
  • coroutine — coroutine support
  • utf8 — UTF-8 string handling

Scripts are isolated from each other — each gets its own Lua environment.

The API object

The api object passed to setup() is organized into namespaces:

Namespace Purpose
api.irc.* Send IRC messages (say, action, notice, join, part, etc.)
api.ui.* Print messages, switch buffers, execute commands
api.store.* Read client state (nicks, buffers, connections)
api.config.* Per-script and app config access
api.on/once/off Event handler registration
api.command Register custom slash commands
api.timer/timeout Repeating and one-shot timers
api.log Debug logging

Quick example: dice game

meta = {
    name = "dice",
    version = "1.0",
    description = "!dice game -- roll dice in channel",
}

function setup(api)
    api.on("irc.privmsg", function(ev)
        local cmd, rest = (ev.message or ""):match("^(!dice)%s*(.*)")
        if not cmd then return end

        local count, sides = 1, 6
        if rest ~= "" then
            local c, s = rest:match("^(%d+)d(%d+)$")
            if c then count, sides = tonumber(c), tonumber(s)
            else
                local s2 = rest:match("^d(%d+)$")
                if s2 then sides = tonumber(s2) end
            end
        end

        count = math.max(1, math.min(count, 20))
        sides = math.max(2, math.min(sides, 1000))

        local total = 0
        for i = 1, count do total = total + math.random(1, sides) end
        api.irc.say(ev.channel, ev.nick .. " rolls " .. count .. "d" .. sides .. ": " .. total)
    end)
end
